## REACT-080

### How do you localize React applications?

**Localization (l10n)** is adapting UI content for a specific locale — translating strings, formatting dates/numbers/currencies, adjusting text direction (LTR/RTL).

**Core steps:**

**1. Choose a library**
- `react-intl` (FormatJS) — full ICU message format, date/number/plural support.
- `i18next` + `react-i18next` — most popular, plugin ecosystem.
- `react-i18n` — lightweight alternative.
- Native `Intl` API — for formatting only, no translation layer.

**2. Structure translation files**
```json
// locales/en.json
{
  "greeting": "Hello, {name}!",
  "itemCount": "{count, plural, one {# item} other {# items}}",
  "nav.home": "Home",
  "nav.about": "About"
}

// locales/es.json
{
  "greeting": "¡Hola, {name}!",
  "itemCount": "{count, plural, one {# elemento} other {# elementos}}",
  "nav.home": "Inicio",
  "nav.about": "Acerca de"
}
```

**3. Provide locale context at the app root**
```jsx
import { IntlProvider } from 'react-intl';
import enMessages from './locales/en.json';
import esMessages from './locales/es.json';

const messages = { en: enMessages, es: esMessages };

function App() {
  const [locale, setLocale] = useState('en');

  return (
    <IntlProvider locale={locale} messages={messages[locale]}>
      <LocaleSwitcher onSwitch={setLocale} />
      <MainApp />
    </IntlProvider>
  );
}
```

**4. Use translations in components**
```jsx
import { FormattedMessage, useIntl } from 'react-intl';

function Header({ userName }) {
  return (
    <h1>
      <FormattedMessage id="greeting" values={{ name: userName }} />
    </h1>
  );
}
```

**Best practices:**
- Never hard-code user-facing strings — always use a message key.
- Use ICU message format for pluralization and gender to avoid manual branching.
- Lazy-load locale files to avoid bundling all translations upfront.
- Test RTL layouts (Arabic, Hebrew) with `dir="rtl"` on the root element.

## REACT-081

### What is `react-intl`?

**`react-intl`** is part of the **FormatJS** library suite — a widely used React internationalization library that provides components and hooks for translating strings, formatting dates, times, numbers, and plurals using the industry-standard **ICU Message Format**.

**Install:**
```bash
npm install react-intl
```

**Key parts:**
- `<IntlProvider>` — provides locale and messages to the component tree via context.
- `<FormattedMessage>` — renders a translated, interpolated message.
- `<FormattedDate>`, `<FormattedTime>`, `<FormattedNumber>` — locale-aware formatting components.
- `useIntl()` — hook to access the intl object for imperative formatting (e.g., in `placeholder` attributes, `aria-label`).
- ICU message format — supports variables, plurals, selects, rich text.

**Minimal example:**
```jsx
import { IntlProvider, FormattedMessage } from 'react-intl';

const messages = {
  en: { 'hello': 'Hello, {name}!' },
  fr: { 'hello': 'Bonjour, {name}!' },
};

function App({ locale = 'en' }) {
  return (
    <IntlProvider locale={locale} messages={messages[locale]}>
      <FormattedMessage id="hello" values={{ name: 'Akar' }} />
    </IntlProvider>
  );
}
```

## REACT-082

### What are the main features of `react-intl`?

**1. String translation with ICU message format:**
```jsx
// Supports variables, plurals, selects, and rich text inline
<FormattedMessage
  id="cart.summary"
  values={{
    count: 3,
    strong: chunks => <strong>{chunks}</strong>,
  }}
/>
// Message: "You have <strong>{count, plural, one {# item} other {# items}}</strong> in your cart."
```

**2. Number formatting:**
```jsx
<FormattedNumber value={1234567.89} style="currency" currency="USD" />
// → $1,234,567.89 (locale-aware)

<FormattedNumber value={0.85} style="percent" />
// → 85%
```

**3. Date and time formatting:**
```jsx
<FormattedDate value={new Date()} year="numeric" month="long" day="2-digit" />
// → June 22, 2026 (in en-US)
```

**4. Relative time:**
```jsx
<FormattedRelativeTime value={-3} unit="day" />
// → 3 days ago
```

**5. Plural rules:**
```
// ICU: {count, plural, zero {No items} one {# item} other {# items}}
```

**6. Select (gender/enumeration):**
```
// ICU: {gender, select, male {He} female {She} other {They}} sent you a message.
```

**7. Rich text (embedded JSX):**
Supports passing React components as values for inline formatting without dangerouslySetInnerHTML.

**8. `useIntl` hook — imperative access:**
```jsx
const intl = useIntl();
const label = intl.formatMessage({ id: 'search.placeholder' });
return <input placeholder={label} />;
```

## REACT-083

### What are the two ways of formatting in `react-intl`?

**Way 1 — React components (declarative):**
Use `<FormattedMessage>`, `<FormattedDate>`, `<FormattedNumber>`, `<FormattedTime>`, `<FormattedRelativeTime>` directly in JSX.

```jsx
import { FormattedMessage, FormattedDate, FormattedNumber } from 'react-intl';

function Invoice({ amount, dueDate }) {
  return (
    <div>
      <p>Amount: <FormattedNumber value={amount} style="currency" currency="USD" /></p>
      <p>Due: <FormattedDate value={dueDate} dateStyle="long" /></p>
      <p><FormattedMessage id="invoice.note" values={{ amount }} /></p>
    </div>
  );
}
```

**Way 2 — `useIntl` hook (imperative):**
Call formatting methods on the `intl` object returned by `useIntl()`. Use when you need the formatted value as a plain string — e.g., for HTML attributes or third-party components.

```jsx
import { useIntl } from 'react-intl';

function SearchBar() {
  const intl = useIntl();

  return (
    <input
      type="search"
      placeholder={intl.formatMessage({ id: 'search.placeholder' })}
      aria-label={intl.formatMessage({ id: 'search.ariaLabel' })}
    />
  );
}
```

**Available imperative methods on `intl`:**
- `intl.formatMessage({ id, defaultMessage }, values)`
- `intl.formatDate(value, options)`
- `intl.formatTime(value, options)`
- `intl.formatNumber(value, options)`
- `intl.formatRelativeTime(value, unit, options)`
- `intl.formatPlural(value, options)`

**When to use which:**
- Rendering text inside JSX → use components.
- Passing a string to a prop, attribute, or non-React API → use `useIntl`.

## REACT-084

### How do you use `FormattedMessage` as a placeholder using `react-intl`?

`<FormattedMessage>` renders JSX — it cannot be directly used as a `placeholder` string attribute. Use `useIntl().formatMessage()` instead.

```jsx
import { useIntl } from 'react-intl';

function SearchInput() {
  const intl = useIntl();

  return (
    <input
      type="search"
      // formatMessage returns a plain string — safe for HTML attributes
      placeholder={intl.formatMessage({
        id: 'search.placeholder',
        defaultMessage: 'Search products...',
      })}
      aria-label={intl.formatMessage({ id: 'search.ariaLabel' })}
    />
  );
}
```

**With variables in the placeholder:**
```jsx
placeholder={intl.formatMessage(
  { id: 'search.placeholderWithCount', defaultMessage: 'Search {count} products...' },
  { count: totalProducts }
)}
```

**Why not use `<FormattedMessage>`?**
`<FormattedMessage>` returns a React element, not a string. Passing a React element to `placeholder` results in `[object Object]` being displayed.

**Alternative — `defineMessages` for type-safety:**
```jsx
import { defineMessages, useIntl } from 'react-intl';

const messages = defineMessages({
  placeholder: { id: 'search.placeholder', defaultMessage: 'Search...' },
});

function Search() {
  const intl = useIntl();
  return <input placeholder={intl.formatMessage(messages.placeholder)} />;
}
```

## REACT-085

### How do you access the current locale with React Intl?

Use the `useIntl` hook — the returned `intl` object exposes the current `locale` string.

```jsx
import { useIntl } from 'react-intl';

function LocaleDisplay() {
  const intl = useIntl();

  return (
    <div>
      <p>Current locale: {intl.locale}</p>
      {/* Use locale for conditional rendering */}
      {intl.locale.startsWith('ar') && <RTLWarning />}
    </div>
  );
}
```

**Accessing locale via `useIntl` in a utility:**
```jsx
function DateFormatter({ date }) {
  const { locale } = useIntl();

  // Use the locale directly with the native Intl API
  const formatted = new Intl.DateTimeFormat(locale, {
    weekday: 'long',
    year: 'numeric',
    month: 'long',
    day: 'numeric',
  }).format(date);

  return <time dateTime={date.toISOString()}>{formatted}</time>;
}
```

**Building a locale switcher:**
```jsx
// The locale is owned by the parent that renders <IntlProvider>
function LocaleSwitcher({ currentLocale, onSwitch }) {
  const locales = [
    { code: 'en', label: 'English' },
    { code: 'es', label: 'Español' },
    { code: 'fr', label: 'Français' },
  ];

  return (
    <select value={currentLocale} onChange={e => onSwitch(e.target.value)}>
      {locales.map(l => (
        <option key={l.code} value={l.code}>{l.label}</option>
      ))}
    </select>
  );
}
```

**Persisting locale choice:** Save the selected locale to `localStorage` and read it on app init to restore the user's preference.

## REACT-086

### How do you format dates using `react-intl`?

**Method 1 — `<FormattedDate>` component:**
```jsx
import { FormattedDate } from 'react-intl';

// Short date
<FormattedDate value={new Date()} />
// → 6/22/2026 (in en-US)

// Long date with custom options
<FormattedDate
  value={new Date('2026-06-22')}
  weekday="long"
  year="numeric"
  month="long"
  day="2-digit"
/>
// → Monday, June 22, 2026

// Predefined style
<FormattedDate value={new Date()} dateStyle="full" />
// → Monday, June 22, 2026
```

**Method 2 — `intl.formatDate()` for string output:**
```jsx
const intl = useIntl();

const dateString = intl.formatDate(new Date(), {
  year: 'numeric',
  month: '2-digit',
  day: '2-digit',
});
// → "06/22/2026"

// As a time attribute value
return (
  <time dateTime={date.toISOString()}>
    {intl.formatDate(date, { dateStyle: 'long' })}
  </time>
);
```

**Formatting a date in a message string:**
```jsx
// Message: "Last updated: {date}"
<FormattedMessage
  id="lastUpdated"
  values={{
    date: <FormattedDate value={lastUpdated} dateStyle="medium" />,
  }}
/>
```

**Locale-aware output examples:**

| Locale | `dateStyle: "long"` output |
|---|---|
| `en-US` | June 22, 2026 |
| `en-GB` | 22 June 2026 |
| `de-DE` | 22. Juni 2026 |
| `ja-JP` | 2026年6月22日 |

The `Intl.DateTimeFormat` options accepted by `<FormattedDate>` match the ECMA-402 standard — `year`, `month`, `day`, `weekday`, `hour`, `minute`, `second`, `timeZone`, `dateStyle`, `timeStyle`.
