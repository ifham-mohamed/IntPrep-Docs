# Next.js Internationalization (i18n)

This file covers internationalization in Next.js: built-in i18n routing, next-i18next, the useTranslation hook, and App Router i18n patterns.

---

## NEXT-059

### How do you handle internationalization (i18n) in Next.js?

**Pages Router — built-in i18n routing:**
```js
// next.config.js
module.exports = {
  i18n: {
    locales: ['en', 'fr', 'es'],
    defaultLocale: 'en',
    // domains: [{ domain: 'example.fr', defaultLocale: 'fr' }]
  },
};
```
Next.js automatically handles locale detection via Accept-Language header and routes `/fr/about` → `pages/about.tsx` with `locale = 'fr'`.

**App Router — manual locale routing:**
The App Router does NOT have built-in i18n. The recommended pattern is a `[lang]` dynamic segment:

```
app/
  [lang]/
    layout.tsx      ← receives lang param, sets html lang attribute
    page.tsx
    about/
      page.tsx
```

```tsx
// app/[lang]/layout.tsx
import { getDictionary } from '@/lib/i18n';
export default async function LangLayout({ children, params }: { children: React.ReactNode; params: { lang: string } }) {
  const dict = await getDictionary(params.lang);
  return <html lang={params.lang}><body>{children}</body></html>;
}
```

Use `middleware.ts` for locale detection and redirect. Libraries like `next-intl` handle this cleanly.

## NEXT-066

### How do you configure `next-i18next` in Next.js?

`next-i18next` wraps `i18next` and `react-i18next` for seamless Pages Router integration:

```bash
npm install next-i18next react-i18next i18next
```

```js
// next-i18next.config.js
module.exports = {
  i18n: {
    defaultLocale: 'en',
    locales: ['en', 'fr', 'de'],
  },
};
```

```js
// next.config.js
const { i18n } = require('./next-i18next.config');
module.exports = { i18n };
```

```json
// public/locales/en/common.json
{ "welcome": "Welcome", "greeting": "Hello, {{name}}!" }
```

```tsx
// pages/_app.tsx
import { appWithTranslation } from 'next-i18next';
export default appWithTranslation(App);

// pages/index.tsx
import { serverSideTranslations } from 'next-i18next/serverSideTranslations';
export const getStaticProps = async ({ locale }) => ({
  props: { ...(await serverSideTranslations(locale, ['common'])) },
});
```

**Note:** `next-i18next` is designed for the Pages Router. For the App Router, use `next-intl` instead.

## NEXT-073

### What is the `useTranslation` hook in Next.js?

`useTranslation` is from `react-i18next` (used by `next-i18next`) and provides translation functions and i18n state to Client Components:

```tsx
import { useTranslation } from 'next-i18next';

export default function Header() {
  const { t, i18n } = useTranslation('common');

  return (
    <header>
      <h1>{t('welcome')}</h1>
      <p>{t('greeting', { name: 'Akar' })}</p>  {/* interpolation */}
      <p>Current locale: {i18n.language}</p>
      <button onClick={() => i18n.changeLanguage('fr')}>Français</button>
    </header>
  );
}
```

`t(key)` looks up the translation key in the loaded namespace. `i18n.changeLanguage()` switches locale client-side.

In the **App Router** (with `next-intl`), the equivalent is:
```tsx
import { useTranslations } from 'next-intl';
const t = useTranslations('common');
// t('welcome')
```

## NEXT-121

### How do you handle i18n in the Next.js App Router?

The App Router has no built-in i18n routing. The standard pattern uses a `[locale]` path segment + `next-intl`:

```bash
npm install next-intl
```

```ts
// middleware.ts
import createMiddleware from 'next-intl/middleware';
export default createMiddleware({
  locales: ['en', 'fr', 'de'],
  defaultLocale: 'en',
});
export const config = { matcher: ['/((?!api|_next|.*\\..*).*)'] };
```

```
app/
  [locale]/
    layout.tsx
    page.tsx
messages/
  en.json
  fr.json
```

```tsx
// app/[locale]/layout.tsx
import { NextIntlClientProvider } from 'next-intl';
import { getMessages } from 'next-intl/server';

export default async function LocaleLayout({ children, params: { locale } }) {
  const messages = await getMessages();
  return (
    <html lang={locale}>
      <body>
        <NextIntlClientProvider messages={messages}>
          {children}
        </NextIntlClientProvider>
      </body>
    </html>
  );
}
```

```tsx
// Server Component
import { useTranslations } from 'next-intl';
const t = useTranslations('HomePage');
return <h1>{t('title')}</h1>;
```

`next-intl` supports Server Components directly, unlike `next-i18next` which is client-focused.
